# 数据集验证指南

## 🔍 在SSH终端执行验证

```bash
cd ~/ppg_project

# 运行验证脚本
python3 validate_dataset.py --data_dir ~/ppg_training_data
```

## 📊 验证内容

脚本会检查：

1. **文件统计** - .npz和.json文件数量
2. **文件完整性** - 检查文件是否损坏
3. **类别分布** - 脉搏类型、心律、伪影分布
4. **信号质量** - 信号长度、均值、标准差
5. **数据充足性** - 评估样本数量是否足够
6. **类别平衡** - 检查各类别样本是否均衡

## ✅ 数据量标准

| 样本数 | 评估 | 说明 |
|--------|------|------|
| < 1,000 | ❌ 不足 | 无法训练 |
| 1,000-5,000 | ⚠️ 较少 | 可训练但效果一般 |
| 5,000-10,000 | ✓ 可以 | 足够训练 |
| > 10,000 | ✓✓ 充足 | 预期高准确率 |

## 🎯 预期结果

**如果生成了20,000样本**：
- ✓ 数据量充足
- ✓ 类别平衡
- ✓ 可以开始训练
- 预期准确率: 95-98%

**如果样本较少**：
```bash
# 生成更多样本
sbatch slurm_datagen.sh  # 再次提交作业
```

## 📈 根据验证结果决定

**验证通过** → 开始训练
```bash
sbatch slurm_train.sh
```

**需要更多数据** → 重新生成
```bash
# 修改样本数量
nano slurm_datagen.sh  # 改 --num_samples 20000 为更大值
sbatch slurm_datagen.sh
```
